Publishing Sphinx Documentation on GitHub Pages

If you prefer writing technical documentation in ReStructured Text (RST) using Python’s Sphinx, you’re in good company. This guide walks you through how to configure and publish your Sphinx documentation on GitHub Pages.

Step 1: Set Up Your Sphinx Documentation

Before you publish to GitHub Pages, you need to generate your Sphinx documentation. Here's how:

$ sphinx-quickstart

Follow the prompts to configure your Sphinx project. Once you have everything set up, you can build the documentation using:

$ make html

        def hello_world():
            print("Hello, world!")

Step 2: Create a GitHub Repository

Create a repository on GitHub where you will host your documentation. If you want the URL to be <username>.github.io, name the repository accordingly. Otherwise, you can name it anything, and GitHub Pages will publish the content to a subdirectory.

Step 3: Add the /docs Folder

For GitHub Pages to serve your Sphinx-built documentation, follow these steps:

  • Add a /docs directory to the master branch of your repository.
  • Go to the repository’s Settings tab and scroll down to the GitHub Pages section.
  • In the Source dropdown, select the master branch /docs folder option.

Step 4: Create a .nojekyll File

Since GitHub Pages uses Jekyll by default, you need to add a .nojekyll file in the /docs folder to prevent Jekyll processing. To do this:

$ touch docs/.nojekyll
                $ git add docs/.nojekyll
                $ git commit -m "Add .nojekyll file"
                $ git push origin master

Step 5: Configure Sphinx for GitHub Pages

In your Sphinx conf.py file, set the output directory for HTML builds to the /docs folder:

import os
                import sys
                sys.path.insert(0, os.path.abspath('.'))
                html_static_path = ['_static']
                html_theme = 'alabaster'
                html_output = 'docs'

Now, you can build your HTML files and place them in the /docs directory:

$ make html
                $ cp -r _build/html/. ../docs

Step 6: Push Changes and Publish

Once everything is set up, push your changes to GitHub:

$ git add .
                $ git commit -m "Publish Sphinx documentation"
                $ git push origin master

After pushing the changes, GitHub Pages will automatically publish the content in the /docs folder to the URL: <username>.github.io/<repository>.

Step 7: Preview and Update

To preview your site, visit the URL provided by GitHub Pages. Make sure everything is working as expected, and continue to update and republish as needed.

Happy documenting!
